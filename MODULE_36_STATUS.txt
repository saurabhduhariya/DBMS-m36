1. WHAT IS MODULE 36?
---------------------
A system that finds patients with similar clinical profiles using pure DBMS
concepts (MongoDB Aggregation Pipelines). NO machine learning libraries used.
Algorithm: Weighted Euclidean Distance computed entirely inside MongoDB.

4. HOW TO RUN THE PROJECT
--------------------------

  PREREQUISITES:
  • Python 3.10+
  • MongoDB Atlas password (ask Saurabh for it)

  STEP-BY-STEP:

  1. Clone the repo and cd into it:
     $ cd Frontend

  2. Create and activate virtual environment:
     $ python3 -m venv venv
     $ source venv/bin/activate          # Linux/Mac
     $ source venv/bin/activate.fish     # Fish shell
     $ venv\Scripts\activate             # Windows

  3. Install dependencies:
     $ pip install -r requirements.txt

  4. Set up .env file (in the Frontend/ root):
     Create a file called .env and paste the following:
     ────────────────────────────────
     MONGO_URI=mongodb+srv://kkpncc:<PASSWORD>@cluster0.yop9wgv.mongodb.net/
     MONGO_DB=medicare_module36
     ────────────────────────────────
     ⚠️  Replace <PASSWORD> with the password shared by Saurabh.
     ⚠️  DO NOT commit .env to Git!
     ⚠️  The .env file is already in .gitignore.

  5. Run the Streamlit app:
     $ streamlit run app.py

  6. Open browser at http://localhost:8501

  7. Navigate to Module 36:
     • Patient View: Sidebar → "F - Case-Based Decision Support" → Click F6

  8. Initialize the database:
     • Go to the 🏠 Home tab
     • Click "🗄️ Initialize Database" button
     • This seeds 20 patients + creates all indexes

  9. Test similarity search:
     • Go to 🔍 SQL Query tab
     • Select a source patient
     • Adjust weight sliders (age, severity, gender, blood group)
     • Click "🚀 Find Similar Patients"
     • Results appear ranked by similarity score

  10. Test feedback:
      • After running a similarity search
      • Scroll to the feedback form
      • Select a match, rate usefulness, add comments
      • Submit — confidence score auto-updates (trigger equivalent)
